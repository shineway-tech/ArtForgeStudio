# Native Client Migration

本说明适用于从旧版“客户端直连供应商”版本迁移到当前“ArtForge 服务端权威”版本。

## 会保留的数据

- 本地生成作品及其元数据。
- 提示词草稿、主题、语言、卡片样式和工作区设置。
- 成功登录后写入系统凭据库的刷新会话；访问令牌只保存在内存。
- 未完成生成任务和待支付订单的恢复记录。

## 不再采用的数据

- 旧手机号、固定验证码、本地登录状态。
- 旧本地积分、每日积分、充值记录和邀请返利数据。
- 旧 Provider Endpoint、API Key、模型测速结果和供应商模型选择。

客户端读取旧 `local-store.json` 时会忽略 Provider 和 API Key，只迁移作品、草稿与界面数据；
下一次正常保存会写入不含 Endpoint/API Key 的新结构。旧密钥不会上传到 ArtForge 服务端，也不会
重新出现在界面中。若旧版本还安装在其他目录，请在确认不再使用后，通过旧版本删除 Provider，
或直接删除该旧版本自己的 `local-store.json`；不要删除当前客户端的作品目录。

## 首次启动

1. 使用邮箱验证码重新登录，并接受当前版本的用户协议与隐私政策。
2. 核对服务端会员、积分、模型目录和设备会话。
3. 在“我的资产”确认旧作品仍可浏览。
4. 如存在未完成订单或生成任务，保持联网，客户端会按原请求 ID 恢复，不会重复创建业务。

账号、会员、积分、订单、支付结果、模型价格和生成任务状态均以服务端为准。本地缓存不具备修改
这些事实的权限。
