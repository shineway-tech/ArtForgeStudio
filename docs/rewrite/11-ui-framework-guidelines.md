# ArtAIT Rust 重构资料：界面框架规范

结论：ArtAIT 的主界面应稳定为 **桌面生产工具框架**，而不是营销页、卡片堆叠页或多弹窗工具。框架以"左侧导航 + 顶部上下文 + 左侧输入面板 + 右侧结果工作区 + 分层浮层"为基础，所有新功能必须先落到明确区域，再考虑视觉细节。

## 设计目标

- 点开软件后立即进入可操作界面，避免额外启动页、跳转页和风格不一致的加载界面。
- 用户的主路径保持短：输入提示词、添加参考图、选择参数、生成、查看结果、复用结果。
- 所有页面共享同一套骨架，减少每个功能页各自长出一套布局。
- 创作页优先保证高频操作效率，视觉装饰让位于信息密度和稳定对齐。
- 功能增加时不破坏主框架，只通过抽屉、浮层、面板分区扩展。

## 全局布局

主窗口分为 5 个固定职责区域：

```text
┌────────────┬────────────────────────────────────────────┐
│            │ 顶部上下文栏：页面标题、模型选择、全局动作 │
│ 左侧导航栏 ├──────────────┬─────────────────────────────┤
│            │ 输入参数面板 │ 结果工作区 / 图库 / 预览区   │
│            │              │                             │
└────────────┴──────────────┴─────────────────────────────┘
```

### 左侧导航栏

职责：

- 只放页面入口、功能模块入口、系统入口。
- 支持展开和最小化，最小化时必须保留可识别 SVG 图标。
- 记录用户上次展开状态，下次启动恢复。

不放：

- 当前页面的临时操作。
- 生成按钮、请求列表、历史记录等工作区动作。
- 只在某个页面有效的配置项。

导航分组固定为：

- 欢迎 / 首页
- 创作
- 图库
- 系统

系统组下放"运行日志"、"侧边栏管理"等系统级入口。底部只保留全局工具图标，不再放重复文字入口。

### 顶部上下文栏

职责：

- 展示当前页面上下文。
- 放全局 provider / 模型选择。
- 放少量全局动作，如设置、主题。

规则：

- 顶栏控件高度统一，图标、文字、下拉箭头垂直居中。
- 下拉箭头不能使用手写偏移；必须由统一 Select 组件负责布局。
- 顶栏不承载当前任务的详细状态，任务状态进入请求列表或底部状态栏。

### 输入参数面板

职责：

- 承载当前页面的全部输入：提示词、参考图、比例、品质、数量、生成按钮。
- 提示词相关动作必须靠近提示词输入框。
- 页面级临时浮层从这里触发，例如历史记录、提示词优化结果。

优先级：

1. 提示词输入
2. 参考图
3. 生成参数
4. 生成 / 取消
5. 历史记录 / 请求列表 / 优化结果

规则：

- 输入面板宽度保持稳定，不随右侧内容变化。
- 生成按钮是主动作，视觉权重最高。
- 取消生成是次动作，不能和主动作使用同等强调色。
- 提示词优化完成后的结果提示只短暂停留，默认 3 秒后自动收起。

### 结果工作区

职责：

- 默认展示当前页面生成结果图库。
- 支持图片预览、元信息、打开位置、使用提示词、删除等结果操作。
- 承载请求列表抽屉，但不改变输入面板宽度。

规则：

- 缩略图默认不显示文件名，避免视觉噪声。
- 缩略图 hover 时显示模型名和品质，必要时再显示短文件名。
- 结果卡片选中、hover、右键菜单状态必须清晰区分。
- 大图预览使用统一预览层，顶部显示文件名和基础尺寸，右侧显示元信息，底部放操作按钮。

### 底部状态栏

职责：

- 显示轻量状态：就绪、任务提交、保存完成、错误摘要。
- 不显示长日志。
- 不承担主要导航。

长日志进入"运行日志"页面。

## 弹层层级

界面只允许 4 类浮动层级。

### 小浮层

适用：

- 下拉选择
- 提示词历史
- 右键菜单
- tooltip

规则：

- 依附触发控件。
- 失去焦点或点击外部关闭。
- 不遮挡主路径按钮。
- 不用于展示大量任务或长文本。

### 侧边抽屉

适用：

- 请求列表
- 当前页面批量记录
- 可滚动的辅助管理信息

规则：

- 从结果工作区左侧或右侧覆盖弹出。
- 高度与结果图库一致。
- 不压缩输入面板。
- 打开后仍能看见当前页面上下文。

### 模态窗口

适用：

- 元信息查看
- 确认删除
- 设置中的危险操作确认

规则：

- 必须有明确关闭动作。
- 不承载普通浏览行为。
- 不和侧边抽屉同时抢焦点。

### 全屏预览层

适用：

- 图片大图预览
- 视频预览

规则：

- 背景压暗。
- 图片区域优先，元信息作为右侧固定栏。
- 底部操作按钮高度紧凑，主次明确。
- 文件名、尺寸、路径排版固定，不能挤压图片主体。

## 组件尺寸规范

为避免图标、文字、箭头错位，所有基础组件必须统一高度和内边距。

| 组件 | 高度 | 圆角 | 说明 |
|------|------|------|------|
| 顶栏 Select | 30px | 7px | 模型选择、provider 选择 |
| 普通按钮 | 30px | 7px | 清空、请求列表、打开等 |
| 小按钮 | 26px | 6px | chip、列表过滤 |
| 主按钮 | 54px | 9px | 开始生成 |
| 图标按钮 | 32px | 7px | 顶栏、侧栏底部工具 |
| 输入框 | 34px | 8px | 单行输入 |
| 文本域 | 页面自适应 | 9px | 提示词输入 |
| 卡片 | 视内容 | 8px | 请求行、图库项 |

对齐规则：

- 图标统一 16px 或 18px，功能图标来自 `ui/assets/icons/`。
- 文字和图标使用同一垂直中心线。
- 下拉箭头固定 10px 到 12px，右内边距由 Select 组件统一处理。
- 按钮内文字不得靠手写 `x/y` 微调。
- 同一行控件高度必须一致。

## 颜色与视觉密度

整体气质：安静、专业、工具化。

规则：

- 深色和浅色主题都必须使用同一布局密度。
- 背景层级最多 4 层：窗口背景、面板背景、悬浮背景、选中背景。
- 卡片圆角不超过 8px，预览层和抽屉可使用 10px 到 12px。
- 边框优先于重阴影；阴影只用于抽屉和模态层。
- 主强调色只用于当前选中、主按钮、成功状态中的关键提示。
- 不使用装饰性渐变、光斑、营销式大 hero。

## 状态反馈

所有创作相关操作必须覆盖以下状态：

- 空状态：无结果、无请求、无历史记录时给出短文本。
- 加载中：明确展示当前阶段，例如上传、生成、轮询、保存。
- 成功：结果进入图库，并在状态栏给短提示。
- 失败：显示可读错误，不泄露 API Key。
- 取消：请求进入取消状态，不伪装成失败。

请求列表显示：

- 当前页面请求优先。
- 缩略图可用时显示缩略图，不可用时显示占位图标。
- 状态 chip 使用固定文案：进行、完成、失败、取消、等待、保存。
- 清空动作必须区分全部、完成、失败。

## 信息架构规则

新增功能时先判断归属：

| 功能类型 | 放置位置 |
|----------|----------|
| 页面入口 | 左侧导航栏 |
| 当前页面输入 | 输入参数面板 |
| 当前页面辅助记录 | 小浮层或侧边抽屉 |
| 生成结果 | 结果工作区 |
| 文件操作 | 结果卡片右键菜单或预览底部 |
| 系统配置 | 设置页 |
| 长日志 | 运行日志页 |
| 全局模型 | 顶部上下文栏 |

若一个功能找不到明确位置，先不要加入口，应先补充本规范或页面规格。

## 启动体验

启动体验遵循：

- 优先直接进入主界面。
- 不使用第二套风格的启动窗口。
- 如必须等待，使用主界面内轻量状态反馈。
- 恢复上次页面时，不先跳首页再切换，应直接加载目标页面状态。
- 启动日志默认不刷屏到命令行，用户需要时在运行日志页查看。

## 实现约束

- Slint 中新增功能组件前，优先复用基础组件。
- 组件布局优先使用 `HorizontalLayout` / `VerticalLayout`，少用手写坐标。
- 必须手写坐标时，只用于预览层、画布、覆盖层等固定结构。
- 所有功能图标使用 SVG，禁止 emoji 或纯文字代替图标。
- Rust callback 不放复杂 UI 业务，仍按 app → service 的边界下沉。

## 验收标准

任一 UI 改动完成前至少检查：

1. 左侧导航、顶栏、输入面板、结果区职责没有混用。
2. 控件图标、文字、下拉箭头在同一行内垂直居中。
3. 小浮层、抽屉、模态层没有互相遮挡关键操作。
4. 缩小窗口后文本不重叠、不挤出按钮。
5. 深色和浅色主题下边框、选中、hover 状态都可辨识。
6. 空、加载、成功、失败、取消状态有明确反馈。
7. 新功能入口能被用户在 2 步内找到。

## 后续执行顺序

1. 统一基础组件尺寸：Button、Select、IconButton、Input。
2. 清理手写偏移导致的图标和箭头错位。
3. 统一请求列表、历史记录、元信息、大图预览的层级样式。
4. 把页面规格中仍然含糊的入口归属补齐。
5. 再进入更细的颜色、动效、字体和主题精修。
