# ArtAIT V1.5 产品设计草案：游戏资产导演级控制

本文档定义 ArtAIT 下一版本 V1.5 的产品方向与界面控制模型。V1.5 的重点不是暴露更多模型采样参数，而是在保持低门槛输入的前提下，加入游戏开发者真正需要的用途、视角、氛围和产出目标控制。

## 产品定位

ArtAIT V1.5 面向游戏开发者，而不是通用生图用户。

愿景：

> 打造一款真正懂游戏美术生产流程的 AI 游戏资产生成平台。

产品应从“输入 Prompt 后得到图片”的通用工具，转向“按游戏生产用途生成可落地资产参考”的专业工作台。

## 阶段定义

### V1 已完成

基础生成能力：

- 场景生成
- 角色生成
- 特效生成
- UI 生成

核心流程：

```text
输入 Prompt
↓
点击生成
↓
获得图片
```

优点：

- 简单
- 上手快

缺点：

- 专业控制不足
- 游戏开发特色不明显
- 与 Midjourney、GPT Image 等通用生图工具差异较小

### V1.5 目标

在保持“简单易用”的基础上，加入游戏开发专属控制能力。

核心原则：

> 默认简单，专业可控。

V1.5 不应把专业参数默认堆到用户面前，而是让普通用户继续用低成本入口生成，让专业用户按需展开导演级控制。

## 界面设计原则

### 默认模式

默认模式服务 80% 用户，仅展示最小必要输入：

- Prompt
- 风格
- 用途
- 生成按钮

示例：

```text
Prompt：森林村庄
风格：日系幻想
用途：场景概念图

[生成]
```

适合用户：

- 策划
- 独立开发者
- 非专业美术
- 快速探索阶段的美术或开发者

### 高级模式

高级模式服务 20% 专业用户。默认折叠，通过“展开高级控制”进入。

```text
展开高级控制
```

展开后显示导演级控制项：

- 游戏视角
- 天气
- 时间
- 光照风格
- 色彩氛围
- 构图
- 细节等级

高级控制不要求用户理解底层模型参数，而是用游戏开发语义表达生成目标。

## 高级控制设计

### 1. 用途

用途是 V1.5 区别于普通生图工具的最高优先级控制项。它决定提示词构建、默认比例、负面约束、输出目录和后续工作流入口。

场景用途：

- 场景概念图
- TileSet
- 地编参考
- 宣传图
- Loading 图
- 小地图
- 建筑套件

角色用途：

- 角色立绘
- 三视图
- 8 方向
- SpriteSheet
- Spine 拆件
- NPC 头像
- 宣传海报

特效用途：

- 技能特效
- Buff 特效
- 爆炸
- 场景特效
- UI 特效
- 武器拖尾

UI 用途：

- HUD
- 主界面
- 背包
- 商城
- Icon
- Loading 界面
- 弹窗

设计要求：

- 默认模式也必须显示“用途”。
- 用途优先于风格、光照、构图等其他控制。
- 不同 `CreationMode` 只显示自身相关用途。

### 2. 游戏视角

游戏视角适用于场景和角色生成，是游戏资产生成比摄影类生图更需要固定控制的关键项。

可选项：

- Top Down
- 2.5D
- Isometric
- Side View
- Third Person
- First Person
- Orthographic（UI）

生成时系统应自动把视角映射为稳定的提示词约束。

价值：

> 游戏开发远比摄影镜头更依赖固定视角。

### 3. 天气与时间

天气与时间用于控制场景氛围，也可以影响 Loading 图、宣传图和 Key Art。

天气：

- 晴天
- 阴天
- 雨天
- 暴风雨
- 雪天
- 雾天
- 沙尘
- 自定义

时间：

- 清晨
- 正午
- 黄昏
- 蓝调时刻
- 深夜

用户真实需求通常是“雨夜村庄”，而不是“CFG=7.5”。因此天气与时间应优先以自然语言意图表达。

### 4. 光照风格

光照风格属于导演级控制，用于把用户可理解的选项映射为专业提示词。

可选项：

- 柔和自然光
- 电影感
- 梦幻发光
- 高对比
- 体积光
- 霓虹灯

示例映射：

```text
电影感
↓
dramatic lighting
high contrast
cinematic composition
volumetric lighting
```

用户无需理解底层英文提示词，系统负责做可解释映射。

### 5. 色彩氛围

色彩氛围本质上是 Color Grading 预设，用于控制整体情绪。

可选项：

- 温暖治愈
- 冷峻压抑
- 高饱和
- 低饱和
- 暗黑风
- 赛博朋克
- 日系幻想
- 吉卜力风

设计要求：

- 默认模式中的“风格”可以优先复用色彩氛围。
- 高级模式中允许用户同时指定风格和色彩氛围，但应避免重复或冲突提示词。

### 6. 构图

构图主要用于宣传图、Loading 图、Key Art、角色海报和 UI 概念图。

可选项：

- 居中
- 左偏
- 右偏
- 三分法
- 广角全景
- 特写

### 7. 细节等级

细节等级用于表达生成目标，而不是模型算力参数。

| 等级 | 目标 |
|------|------|
| 草图 | 快速探索 |
| 概念设计 | 设计确认 |
| 游戏美术级 | 生产参考 |
| 宣传海报级 | 营销素材 |

可选项：

- 草图
- 概念设计
- 游戏美术级
- 宣传海报级

## 开发优先级

### P0 必须

- 用途
- 游戏视角
- 色彩氛围
- 天气 / 时间
- 光照风格

### P1 推荐

- 构图
- 细节等级
- 种子固定
- 参考图权重
- 负面提示词

### P2 后续

- 高级 Prompt 编辑器
- Prompt 模板系统
- 项目风格记忆
- 自动扩展工作流

## 不建议默认暴露的参数

以下参数建议隐藏：

- CFG
- Steps
- Sampler
- Scheduler
- Clip Skip
- Eta
- Denoise

原因：

> 游戏开发者真正关心的是“生成什么”，而不是“模型怎么采样”。

这些参数可以放入“开发者模式”，默认关闭。

## 产品差异化

普通生图工具：

```text
Prompt
↓
图片
```

游戏 AI 生图工具：

```text
Prompt
↓
游戏用途
↓
导演控制
↓
符合游戏生产需求的结果
```

V1.5 的用户感知价值：

> 这个工具不是单纯会生图，而是懂游戏美术。

对游戏开发者而言：

```text
生成一个雨夜 2.5D 的 TileSet 场景
```

比：

```text
CFG=7.5、Euler、30 Steps
```

更重要。

游戏开发者更关注生成工具与现有生产流程的契合度、常用导出格式和可控性，而不是底层模型参数。

## 与现有架构的映射

V1.5 不要求重写现有生成链路，应优先在当前 `CreationMode` 和通用创作工作台上扩展结构化控制。

建议新增模型概念：

- `AssetPurpose`：用途
- `GameView`：游戏视角
- `WeatherPreset`：天气
- `TimeOfDayPreset`：时间
- `LightingPreset`：光照风格
- `ColorMoodPreset`：色彩氛围
- `CompositionPreset`：构图
- `DetailLevel`：细节等级

建议服务层职责：

- UI 只收集结构化选项。
- `artait-service` 将结构化选项转为生成请求。
- Prompt 拼接与映射规则放在 service 或独立 prompt builder，不放在 Slint callback。
- provider 仍只接收最终生成请求，不感知 UI 控件细节。

## V1.5 核心理念

> 简单入口 + 导演级控制。

默认人人都会用。专业用户越用越离不开。

V1.5 是产品从“通用生图工具”转向“游戏美术生产工具”的第一步。
