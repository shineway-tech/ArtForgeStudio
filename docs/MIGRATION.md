# Migration

本说明适用于从旧版“客户端直连供应商”版本迁移到当前“ArtForge 服务端权威”版本。

## Retained data

升级时继续保留：

- 本地生成作品及其展示元数据。
- 提示词草稿和自定义提示词。
- 语言、主题、字体、卡片样式、作品路径等界面偏好。
- 成功登录后写入系统安全凭据的刷新会话；Access Token 只保存在内存。
- 未完成生成、结果交付确认和待支付订单的恢复记录。

## Discarded legacy data

当前客户端不再读取或采用：

- 旧手机号、固定验证码和本地登录状态。
- 旧本地积分、每日积分、充值记录和邀请返利模拟数据。
- Provider Endpoint、供应商 API Key、模型测速结果和供应商模型选择。
- 由客户端直接创建的供应商任务和本地扣费结果。

客户端读取旧 `local-store.json` 时会忽略 Provider 和 API Key，只迁移作品、草稿与界面数据。下一次正常保存会写入不含 Endpoint/API Key 的新结构。旧密钥不会上传到 ArtForge 服务端，也不会重新出现在界面中。

如果旧版本仍安装在其他目录，请确认不再使用后，通过旧版本删除 Provider，或只删除旧版本自己的配置文件。不要删除当前客户端的作品目录或 `data/out` 中的作品。

## First start after migration（首次启动）

1. 使用当前支持的登录方式重新登录。
2. 接受当前版本要求的用户协议与隐私政策。
3. 核对服务端会员、积分、模型目录和设备会话。
4. 在“我的资产”确认旧作品仍可浏览。
5. 如存在未完成订单或生成任务，保持联网并等待客户端按原请求 ID 恢复。

账号、会员、积分、订单、支付结果、模型价格和生成任务状态均以服务端为准。本地缓存只用于展示、离线浏览和恢复，不能修改这些事实。

## Support checks

迁移问题优先确认：

- 当前运行的是哪个安装目录中的客户端。
- 旧版配置文件和当前作品目录是否属于同一安装目录。
- 作品文件是否仍存在，而不是只缺少本地展示索引。
- 用户是否已重新登录并完成服务端账户快照同步。
- 恢复记录中的请求是否已经在服务端完成、取消或过期。

不要通过复制旧 Provider 配置、API Key 或本地积分文件来“修复”新客户端。
