# 无限画布交互增强设计

日期：2026-07-23  
状态：已确认  
参考方向：ComfyUI Frontend 交互、LiteGraph 画布行为

## 1. 目标与范围

本轮在现有 Rust + Slint 架构内连续完成三个阶段：

1. 画布基础：选择/平移模式、框选、多选、聚焦与快捷键。
2. 选择操作：批量移动、复制粘贴、删除、完整分组与嵌套。
3. 连线体验：端口高亮、重连、空白处节点搜索和自动连接。

交互参考 ComfyUI Frontend 和 LiteGraph，但不嵌入 WebView，也不直接引入其源码。现有生成流程、服务端数据结构与 API 保持不变。目标规模为 200 个节点、400 条连线。

## 2. 架构与数据模型

`infinite-canvas-page.slint` 继续负责渲染和输入。Rust 侧在现有无限画布回调模块基础上抽出画布操作层，统一处理选择、移动、复制、删除、分组、连线和撤销。

已有 `CanvasHistory` 继续使用，并扩展为事务式历史记录：一次连续用户操作只生成一个快照。例如，拖动多个节点只产生一条撤销记录，鼠标移动期间不反复写历史或本地存储。

`CanvasNoteData` 增加：

- `parent_group_id`：父分组 ID，空字符串表示顶层。
- `z_index`：同层稳定顺序。

分组关系通过父 ID 表达并支持嵌套。所有节点仍保存画布世界坐标，因此加入或离开分组不会造成屏幕跳动。移动分组时，Rust 计算其全部后代并执行批量位移。

状态分层如下：

- 持久状态：节点、连线、父子分组关系和层级顺序。
- 会话状态：多选集合、框选区域、工具模式、连线预览、端口高亮、搜索弹窗。
- 视图状态：缩放和平移，仅参与坐标换算。

新增持久字段提供 serde 默认值，旧版本项目可直接载入。

## 3. 选择、平移与快捷操作

工具栏提供明确的“选择”和“平移”模式，并显示当前模式。

选择模式：

- 在空白处左键拖动创建框选矩形。
- 框选方向不限，节点与矩形相交即进入选择集。
- 点击节点执行单选；按住 `Ctrl` 点击增减选择。
- 拖动任一已选节点时，整个选择集同步移动。
- 点击空白处清空选择。

平移模式：

- 左键拖动画布。
- 任意模式下，中键拖动或按住空格后左键拖动可临时平移。
- 松开后恢复原工具模式。
- 滚轮继续以鼠标位置为锚点缩放，范围保持 5%–500%。

快捷键：

- `Delete` / `Backspace`：删除选择。
- `Ctrl+C` / `Ctrl+V`：复制粘贴，保留选择内部的连线。
- `Ctrl+D`：向右下偏移快速复制。
- `Ctrl+A`：全选画布节点和分组。
- `Ctrl+Z` / `Ctrl+Y`：撤销和重做。
- `F`：聚焦选择；无选择时聚焦全部内容。
- `Escape`：取消框选、连线或关闭节点搜索。

文本输入框或搜索框获得焦点时，复制、删除、全选等快捷键交由输入控件处理。

多选操作条继续采用固定节点锚定方向，允许超出可视区域，不因缩放或临近边缘而翻转。

## 4. 连线、重连与节点搜索

普通节点保留单输入、单输出端口，分组不参与连线。

拖动输出端口时：

- 显示贝塞尔预览线。
- 实时高亮有效目标端口。
- 重复连接、会形成循环或不兼容的端口显示禁用状态。
- 命中范围按缩放换算为固定屏幕尺寸。

输入端口只允许一条上游连线。新连线落到已连接输入时执行原子重连，旧连接替换与新连接建立合并为一次撤销操作。也可从已有输入端口反向拖出并重新选择上游；取消操作时恢复原连接。

从输出端口拖到空白处时，在释放的世界坐标打开节点搜索弹窗。搜索内容包括文本、图片、视频、音频节点，支持中英文名称和关键词。键盘上下键选择，`Enter` 创建，`Escape` 取消。

创建节点后：

- 新节点定位在释放位置。
- 自动连接来源节点。
- 新节点进入选中状态。
- 弹窗保持世界坐标锚定，允许超出可视区域，不因边缘检测改变方向。

本轮不增加服务端端口类型协议。现有普通节点之间均视作通用数据流，继续沿用上游内容传递机制。兼容性判断集中在 Rust 操作层，为以后增加类型端口预留边界。

## 5. 完整分组容器

点击“分组”时：

- 有选择：在所选内容外围按固定留白创建分组，并建立父子关系。
- 无选择：在当前视图中心创建空分组。

增加 `Ctrl+G` 快速分组与 `Ctrl+Shift+G` 解除最外层分组，工具栏也提供相应操作。

分组行为：

- 移动分组会带动所有直接和间接后代。
- 拖动内部单个节点不会移动分组。
- 节点拖动结束后，以其中心点判断归属。
- 同时命中多个分组时，选择面积最小、层级最深的有效分组。
- 拖到所有分组外时解除归属。
- 允许子分组进入父分组，但禁止分组成为自身或自身后代的子项。
- 调整分组尺寸不缩放内部节点。
- 分组最小尺寸为子项包围盒加固定内边距。

删除分组默认仅解除直接子项归属，不删除内部节点。复制分组则复制全部后代和其内部连线，不复制指向分组外的连线。

渲染顺序为：外层分组背景、内层分组背景、连线、普通节点、选择框和浮动操作条。标题、边框与工具条始终锚定所属分组。

## 6. 性能与异常处理

拖动、框选和连线预览只更新 UI 会话状态，鼠标释放后才批量提交 Rust 数据并保存本地文件。节点命中、框选和分组归属使用包围盒计算；连线目标仅检查普通节点端口。

渲染继续使用当前 Slint 结构，减少重复坐标计算和无效属性绑定，不在本轮重写渲染器。

所有操作先校验后提交：

- 节点或连线达到上限时给出状态提示。
- 非法分组嵌套、循环连线、重复连接和无效粘贴不改变现有数据。
- 旧存储缺失新字段时使用默认值。
- 载入数据若存在无效父 ID或分组循环，只解除异常归属，保留节点内容与位置。
