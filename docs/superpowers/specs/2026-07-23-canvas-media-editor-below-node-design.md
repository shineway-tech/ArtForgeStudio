# 无限画布媒体编辑面板下方锚定设计

## 目标

无限画布中的图片、视频和音频节点被选中后，主编辑面板始终显示在节点下方，不因缩放倍数或可视区域剩余空间而翻转到节点上方。

## 定位规则

- 面板横向继续相对节点居中。
- 面板纵向固定为“节点高度 + 随画布缩放的 20px 间距”。
- 不再根据视口底部剩余空间切换到节点上方。
- 当节点靠近画布底部时，允许面板部分或全部超出当前可视区域；用户可通过拖动画布查看。
- 模型选择、图片参数、视频参数和音频参数等二级 `PopupWindow` 继续使用现有的视口边界约束，本次不修改。

## 实现范围

仅调整 `CanvasNodeCard.media-editor-y()` 的主面板纵向定位，不改变节点尺寸、缩放、拖动、画布平移、二级弹窗或生成逻辑。

## 验证

- 添加源码级回归测试，要求 `media-editor-y()` 始终返回节点底边之后的位置。
- 测试应拒绝旧的“空间不足时翻到节点上方”分支。
- 运行无限画布相关测试和完整 `artforge-studio-native` 测试。
- 编译 Slint UI，确认定位表达式合法。
