# 画布节点操作条定位与等分布局设计

## 目标

统一无限画布中文字、图片、视频和音频节点的顶部操作条：

- 操作条始终位于节点上方，不因节点靠近可视区域顶部而翻转到节点内部。
- 操作条可以超出画布可视区域。
- 操作条内部按钮等宽铺满可用背景区域，不在右侧留下未分配空间。
- 保持现有按钮文案、图标、点击行为、操作条总宽度和缩放逻辑不变。

## 根因

`CanvasNodeCard` 中的文字操作条和媒体操作条根据节点的画布 `y` 坐标选择上方或节点内部位置。当节点靠近可视区域顶部时，条件分支把操作条放到节点内部，造成遮挡。

操作条中的 `CanvasMediaAction` 使用固定宽度。各按钮宽度、间距和左右内边距之和没有完全占满操作条宽度，因此右侧出现空白。

## 设计

### 定位

新增共享的操作条纵向定位函数，返回固定的负向缩放偏移：

`-76px * node-scale`

文字操作条和媒体操作条均使用该函数。函数不读取节点的绝对 `y` 坐标或画布可视高度，因此操作条不会再根据边界翻转。

### 按钮布局

保留现有操作条宽度、左右内边距和按钮间距。移除操作条内各 `CanvasMediaAction` 的固定宽度，并为每个按钮设置相同的 `horizontal-stretch: 1`。

由 `HorizontalLayout` 将扣除内边距和间距后的宽度平均分配给按钮：

- 文字节点：4 个按钮等宽。
- 图片节点：3 个按钮等宽。
- 视频节点：4 个按钮等宽。
- 音频节点：3 个按钮等宽。

## 测试

添加源码结构回归测试，验证：

- 操作条纵向定位只使用固定的节点上方偏移。
- 定位逻辑不再依赖 `root.y` 或 `root.viewport-height`。
- 文字与媒体操作条都使用共享定位函数。
- 所有操作条按钮均使用 `horizontal-stretch: 1`。
- 操作条按钮不再声明固定宽度。
